# 総合型選抜YouTube専門講義脚本（sogo-senbatsu-yt-script-jp）

## 鉄則
1. **評価者側の論理から語る。受験生側のテクニックは語らない。** 市場は「書き方のコツ」で飽和している。このチャンネルの存在理由は、採用・投資審査で実際に人と事業を選別してきた側の一次経験から「選ばれる構造」を明かすことにある。テクニック解説に滑り落ちた台本は書き直す。
2. **経営レンズと哲学レンズは毎回両方使う。** 経営レンズだけなら意識高い系ビジネス動画、哲学レンズだけなら抽象論になる。「実務で選別してきた者が、概念を厳密に定義しながら語る」の掛け算が唯一の差別化。片方しか使っていない台本は不合格。
3. **専門用語は逃げずに使い、必ずその場で一行翻訳する。** 視聴者は高校生と保護者。「デューデリジェンス（買収前に相手を徹底的に調べること）」のように、用語を薄めるのではなく翻訳を添える。専門性の証明と理解可能性を両立させる。
4. **入試制度の事実は毎年変わる。記憶で断定しない。** 出願要件・日程・大学の方針は「要出典確認」を台本に明示し、一般化できる評価の構造だけを断定で語る。制度の誤情報は専門路線チャンネルの信頼を一撃で殺す。

## §1　ホワイトスペース探索（総合型選抜市場の空白）

### STEP 1-1　競合地図（この市場のレッドオーシャン）
検索できる環境では「総合型選抜 YouTube」「志望理由書 書き方」「AO入試 面接」で実検索して地図を更新する。定型地図と企画在庫は `references/sogo-theme-bank.md` を読むこと。

| 勢力 | 典型コンテンツ | 飽和度 |
|---|---|---|
| 総合型専門塾系 | 合格実績、書き方講座、大学別対策 | 極高 |
| 合格者体験談系 | 「私はこれで受かった」 | 極高（n=1で再現性なし） |
| 大学・入試情報系 | 制度解説、日程、倍率まとめ | 高（速報戦は個人が負ける） |
| 小論文・面接講師系 | 型の指導、模擬面接 | 高 |
| 探究学習・教育論系 | 探究の意義、教育制度論 | 中（抽象論が多く実務性が低い） |

### STEP 1-2　この話者にしか語れない空白（三領域）
ホワイトスペースは「評価する側・選別する側の一次経験」に集中して存在する。企画は必ず次のいずれかに立地させる：

1. **経営実務の翻訳**：出願プロセスを資本市場・労働市場の選別実務に写像する（志望理由書＝投資ピッチ、活動実績＝トラックレコード、面接＝採用面接＋DD、アドミッションポリシー＝IR資料）。競合上位に「実際に採用・投資してきた側」からの解説はほぼ不在
2. **哲学の道具箱**：「やりたいことがない」を問いの設計問題として扱う、概念（主体性・探究・貢献）の厳密な定義、論証構造の検査。ハウツーの下にある前提を掘る動画は不在
3. **保護者の意思決定**：総合型対策への課金判断、専門塾の選び方をコスト構造とインセンティブから解説（CFOの得意領域で、かつ誰もやらない）

**ホワイトスペース判定基準**（3つ全部で合格。1つでも欠けたら企画を差し替える）：
1. 競合地図の5勢力が同じ切り口でやっていない
2. 話者の一次経験（経営・哲学）が主根拠になる（受け売りの入試情報が主根拠なら不合格）
3. 視聴者の意思決定（出願戦略・課金・時間配分）を実際に変える実務的含意がある

### STEP 1-3　企画候補の提示
候補5本を「仮タイトル＋中心命題＋使用レンズ（経営／哲学の両方の使い所）＋空白である理由」の4点セットで提示し、1本選ばせる。自動選択の指示があれば判定基準の充足度最上位を選ぶ。

## §2　ペルソナ・エンジン（二枚のレンズ）

### STEP 2-1　経営レンズ：写像表から毎回2つ以上使う
| 総合型選抜の要素 | 経営実務への写像 | 語れる実務論点 |
|---|---|---|
| 志望理由書 | 投資家向けピッチ／エクイティストーリー | なぜ他でなくここか（資本の使途）、実績と計画の接続、数字の裏付け |
| 活動実績・ポートフォリオ | トラックレコード | 再現性の証明、盛った実績がDDでどう剥がれるか |
| 面接 | 採用面接＋デューデリジェンス | 構造化面接、逆質問の設計、情報の非対称性 |
| アドミッションポリシー | IR資料・採用要件定義 | 発行体（大学）が本当に欲しい人材の読み解き方 |
| 併願戦略 | ポートフォリオ理論・資本配分 | リスク分散、期待値、撤退基準 |
| 塾・対策講座への課金 | 投資対効果（ROI）と受託側インセンティブ | 成果報酬構造、情報の非対称性の見抜き方 |

### STEP 2-2　哲学レンズ：道具箱から毎回1つ以上使う
- **問いの設計**：「やりたいことは何か」を「どんな問題なら自分は退屈せず考え続けられるか」に置換する等、答えられない問いを答えられる問いに作り替える技術
- **概念の定義**：「主体性」「探究」「貢献」など募集要項の頻出語を厳密に定義し直し、曖昧語のまま書類を書く危険を示す
- **論証の検査**：志望理由の「だから」が本当に接続しているか（動機→行動→志望の推論の飛躍を検出）
- **通念の吟味**：「好きなことを仕事に」「夢を持て」型の通念を一度解体してから、使える形で再構築する

### STEP 2-3　誠実性ゲート（専門路線の生命線）
- 合格保証・合格率を示唆する表現は使わない（実績訴求はjuku-yt-whitespot-script-jpの集客路線でも禁止だが、専門路線では特に致命傷）
- 経営経験の誇張をしない。語れるのは「選別する側の構造」であり「この大学の採点基準」ではない——この限界を台本内で明示する（限界の明示自体が信頼装置になる）
- 入試制度・大学固有情報はすべて「要出典確認」タグを付け、ユーザーに裏取りを促す

## §3　30分講義型構成テンプレート

反常識路線（推理劇型）と異なり、専門路線は**講義型**：命題を先に立て、フレームワークを渡し、ケースで使って見せ、チェックリストで持ち帰らせる。分速300字換算・約9,000字。ALWAYS use this exact template:

```
【0:00-1:30 命題提示】（約450字）
中心命題を言い切る（例：「総合型選抜は受験ではなく採用です」）。
話者の資格を1文で開示（複数社CFO・COO／哲学研究——長い自己紹介は不要、
肩書の羅列より「私は◯◯人の採用面接をしてきた側の人間です」型の実数開示）。
この30分の講義で渡すもの（フレームワーク＋チェックリスト）を予告。

【1:30-4:00 問題の再定義】（約750字）
哲学レンズ：通説の問いがなぜ答えられないかを示し、問いを立て直す。
「書き方を学ぶ前に、評価者が何を見ているかを知らないのは、
面接官を知らずに面接を受けるのと同じ」型の前提転換。

【4:00-12:00 第1部：フレームワーク提示】（約2,400字）
経営レンズの写像を1つ選び、実務の実際を先に語ってから受験に写像する
（例：投資ピッチが通る条件→志望理由書に翻訳）。
図解指示を多用（［図解：〜］）。8分帯に一次経験の具体談を1本。

【12:00-20:00 第2部：ケース適用】（約2,400字）
架空のモデルケース（例：探究テーマが決まらない高2）にフレームワークを
実際に適用して見せる。ビフォー→アフターの書類・回答例を読み上げる。
※実在の生徒・書類は使わない。16分帯に哲学レンズの概念定義を1本。

【20:00-26:00 第3部：実務チェックリスト】（約1,800字）
明日から使える検査項目5〜7個（「志望理由書DDチェックリスト」等）。
保護者向けの含意を1つ入れる（課金判断・関わり方）。
反論処理を1つ（「経営と受験は違うという反論に対して」）。

【26:00-29:00 まとめと限界の明示】（約900字）
命題再掲→フレームワーク復唱→「私が語れるのは選別の構造であって、
各大学の採点基準ではない」と射程の限界を明示（誠実性ゲート）。

【29:00-30:00 CTA】（約300字）
次回講義のオープンループ。自塾・サービスへの導線は概要欄誘導のみ。
```

**リテンション装置**：講義型は中だるみしやすい。各部の冒頭で「現在地と残りの構成」を宣言し、5分間隔で「一次経験の具体談／モデルケースの続き／視聴者への問いかけ」のいずれかを配置する。

## §4　文体規約（専門・実務トーン）

- 一人語り。断定は経験の範囲で強く、制度事実は留保つきで。一文50字以内目安
- 専門用語＋一行翻訳の形式を守る：「エクイティストーリー、つまり『なぜあなたにお金を託すべきか』の物語」
- 数字で語る癖：「多くの」ではなく「私が見てきた◯百件のうち」型（実数はユーザー入力から。無ければ規模感の抽象表現に落とす）
- 説教しない。評価者の構造を淡々と開示する方が、煽りより専門路線では強い
- 画面指示は `［テロップ：〜］［図解：〜］［チェックリスト表示：〜］` を行頭に

## §5　納品形式

1. **企画書パート**：選定ホワイトスペース、中心命題、使用レンズ（経営写像×哲学道具）、要出典確認リスト
2. **台本本体**：§3テンプレート準拠のフル台本
3. **付帯情報**：タイトル案3本（命題型／写像型／問い型）、サムネ文言案3本、概要欄文面（チェックリストの配布導線つき）
4. **公開前の案内**：hasshin-compliance-jpでの最終チェック（特に合格保証・優良誤認まわり）を必ず案内
