# PRIVACY — プライバシー憲法（mac-neural-grid）

## P1. 平文保存しない秘密（§30）

API キー・パスワード・秘密鍵・Cookie・アクセストークン・個人情報・法務文書本文・生徒情報・
顧客情報・医療情報を、CLI 引数・ログ・JSON に平文保存しない。必要なら macOS Keychain を用いる。

- 実装: `security.redact` / `redact_obj` が events・監査・ログ書込の前に自動適用。
- `verify` は監査/イベントに秘密値が残っていないことを検査する。

## P2. 機密は外部へ出さない（§12）

機密文書は明示的な許可がない限り外部 AI API へ送信しない。`model_router` は機密ポリシー下で
外部 API を選ばず、ローカル推論か決定的処理へフォールバックする。`policy_engine.evaluate` は
機密ポリシーでの外部送信を **violation** として実行前に止める。

## P3. スコープ隔離（§33 privacy）

顧客データを別スコープ/別ノードへ混入させない。ノード間で秘密値を複製しない。ジョブごとに
使用モデル・外部送信有無・扱ったファイルを監査に残す（ただし原文は残さない）。

## P4. データ保持

作業ディレクトリの一時データはジョブ終了後に安全に扱う（cancel/rollback で work/ を掃除）。
成果物は checksum つきで管理し、保持方針を運用で定める。

## P5. 外部送信は人間（§15）

顧客・保護者・会員・第三者への送信、外部 API への送信は high_risk として承認ゲートを経る。
本システムは自動で外部送信しない（MVP は送信経路そのものを実装しない）。
